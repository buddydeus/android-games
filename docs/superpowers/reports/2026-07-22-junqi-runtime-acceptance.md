# 军棋模拟器运行验收

**日期：** 2026-07-22

**设备：** Android 36 `android_games_mvp_pad` 模拟器

**应用版本：** 游戏中心 `0.0.3`，军棋包 `0.0.8`

## 验收范围

- 游戏中心从已安装包清单读取军棋名称和 `assets/icon.png`，首页未增加军棋专用展示分支。
- 军棋主界面显示完整棋盘预览、包内 Logo、版本号，以及 `单人模式`、`双人对战`、`退出游戏`。
- 单人模式完成布阵交换、出战、碰子结果确认、后台 AI 应答、最后移动标记和悔棋恢复。
- 双人模式完成红方布阵、蓝方接管、蓝方布阵和红方接管；两次交接页均不挂载棋盘坐标或军衔语义。
- 2560x1800 物理分辨率和 1600x1200 物理分辨率（320 dpi，对应 800x600dp）下均检查军棋菜单及布阵页。
- 在运行中的安装目录临时移走 `assets/board/junqi-board.png`，确认主界面使用 Compose 备用棋盘；随后恢复原文件并关闭模拟器。

## 结果

| 场景 | 结果 | 证据 |
| --- | --- | --- |
| 首页自动发现 | 通过 | 首页显示包提供的 `军棋` 名称和圆形 Logo，游戏中心仍为 `0.0.3` |
| 军棋菜单 | 通过 | 显示 `版本 0.0.8` 和三个统一菜单按钮 |
| 单人布阵 | 通过 | 25 枚己方棋子完整显示，司令与军长可合法交换，随机、重置、出战可用 |
| 暗棋对局 | 通过 | 敌方仅显示棋背；碰子页只显示通用结果 `防守方胜` |
| AI 与悔棋 | 通过 | AI 后台完成应答，亮蓝四角框标记 AI 最后移动；悔棋恢复玩家行动前局面 |
| 双人隐私 | 通过 | 红蓝交接页 UI 树的军衔计数和棋盘坐标计数均为 0 |
| 800x600dp | 通过 | 棋盘和右侧栏保持并排，菜单、布阵、按钮无重叠或裁切 |
| 素材后备 | 通过 | 棋盘 PNG 缺失时切换为完整可辨识的 Compose 棋盘 |

本地截图和 UI Automator XML 保存在 `build/runtime-acceptance/`，该目录属于构建证据，不纳入版本控制。
