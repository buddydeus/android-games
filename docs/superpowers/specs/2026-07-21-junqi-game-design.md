# Android Pad 二国军棋游戏设计

**状态：** 已批准
**日期：** 2026-07-21
**游戏显示名：** 军棋
**游戏包 ID：** `junqi`
**初始版本：** `versionCode = 1`，`versionName = 0.0.1`

## 目标

在现有离线 Android Pad 游戏中心中增加一款可独立构建、导入和更新的二国军棋游戏包。游戏采用标准暗棋规则，支持单人机器人对战和同一设备上的本地双人遮屏交接，不向游戏中心外壳泄露规则、AI、界面或素材实现。

## 架构边界

- 新增 Android Library 模块 `games/junqi`，依赖现有 `game-api`、Compose Foundation、Material 3 和 Compose UI，不增加外部运行时依赖。
- `JunqiPlugin` 实现 `GamePlugin`，主界面显示 `GameManifest.versionName`，菜单文字固定为 `单人模式`、`双人对战`、`退出游戏`。
- 包内容位于 `games/junqi/package/`，构建结果为 `build/game-packages/junqi.zip`。
- zip 顶层包含 `manifest.json`、`plugin.apk` 和 `assets/`。名称与 Logo 由包清单提供，游戏中心继续按成功启动次数排序。
- 将 `junqi.zip` 作为第五个内置包复制进 Debug APK 只改变内置游戏内容，不改变外壳 API、加载逻辑或游戏中心版本。
- 规则状态和真实棋子身份只存在于军棋包内部。AI 入口只能接收观察者可见状态，不能读取真实敌方军衔。

## 规则基线

规则依据安徽省大学生计算机博弈大赛军棋项目规程及中国大学生计算机博弈大赛棋谱标准：

- https://cxcyxy.chnu.edu.cn/xkjs/content_123851
- https://www.manuallib.com/download/338F27602187BCA684E969685677FAEE.pdf

### 棋盘与坐标

- 棋盘为 12 行 5 列共 60 个停靠点，模型行从上到下为 `0..11`，列从左到右为 `0..4`。
- 上方阵营为蓝方，下方阵营为红方。红方先手。
- 大本营坐标为蓝方 `(0,1)`、`(0,3)`，红方 `(11,1)`、`(11,3)`。
- 行营坐标为：
  - 蓝方：`(2,1)`、`(2,3)`、`(3,2)`、`(4,1)`、`(4,3)`。
  - 红方：`(7,1)`、`(7,3)`、`(8,2)`、`(9,1)`、`(9,3)`。
- 其余节点为兵站。空行营可进入；敌方不能攻击已被棋子占据的行营。

### 公路图

- 每一行的相邻列都有水平公路边。
- `0↔1` 与 `10↔11` 行之间的五列均有垂直公路边。
- `1..5` 与 `6..10` 区域只在列 `0`、`2`、`4` 具有相邻行垂直公路边。
- `5↔6` 的中部桥接位于列 `0`、`2`、`4`。
- 每个行营通过对角公路连接其左上、右上、左下、右下节点，并保留同一行左右公路连接。
- 普通公路移动每次只能沿一条边到相邻节点。

### 铁路图

- 水平铁路位于行 `1`、`5`、`6`、`10`，连接该行全部相邻列。
- 垂直铁路位于列 `0`、`4`，连接行 `1..10` 的相邻节点。
- 中央铁路桥连接 `(5,2)` 与 `(6,2)`。
- 路径上不能越过任何棋子。
- 工兵可在铁路图上沿任意连续路径转弯并移动任意距离。
- 其他可移动棋子只能在同一行或同一列的连续铁路线上直行任意距离，不能转弯。

### 棋子集合

每方 25 枚：

| 棋子 | 数量 | 军阶/性质 |
| --- | ---: | --- |
| 司令 | 1 | 最高军阶 |
| 军长 | 1 | 军阶 8 |
| 师长 | 2 | 军阶 7 |
| 旅长 | 2 | 军阶 6 |
| 团长 | 2 | 军阶 5 |
| 营长 | 2 | 军阶 4 |
| 连长 | 3 | 军阶 3 |
| 排长 | 3 | 军阶 2 |
| 工兵 | 3 | 军阶 1，可转弯走铁路并排雷 |
| 地雷 | 3 | 不可移动 |
| 炸弹 | 2 | 与任何敌子同归于尽 |
| 军旗 | 1 | 不可移动，被夺取即失败 |

### 布阵

- 每方在己方六行区域的 25 个兵站和大本营中布置全部 25 枚棋子，行营不能布子。
- 军旗必须放在己方两个大本营之一。
- 地雷只能放在本方最后两排，即蓝方行 `0..1`、红方行 `10..11`。
- 炸弹不能放在第一线，即蓝方行 `5`、红方行 `6`。
- 布阵界面提供一个合法默认阵型、合法随机阵型、重置、点击两枚棋子交换和 `出战`。
- 交换操作只有在交换后仍满足全部位置限制时才生效。
- 单人模式只让玩家布置己方，AI 使用确定性种子生成合法随机阵型。
- 双人模式由红方和蓝方依次在完全遮蔽的交接流程中布阵。

### 移动与碰子

- 军旗、地雷和任何进入大本营的棋子不可移动。
- 其他棋子可按公路或铁路规则移动到空节点，或攻击不在行营中的敌方棋子。
- 普通军阶碰子时高军阶存活，低军阶移出棋盘；相同军阶同归于尽。
- 炸弹与任何敌方棋子碰子时双方移出棋盘，包括地雷和军旗。
- 工兵攻击地雷时工兵存活、地雷移出；地雷遭炸弹攻击时同归于尽；其他棋子攻击地雷时攻击方移出、地雷保留。
- 军旗不能攻击；军旗被任何可成功到达的大子夺取后，进攻方获胜。
- 司令被吃或同归于尽后，该方军旗位置对双方永久公开。
- 碰子只公开 `进攻方胜`、`防守方胜` 或 `同归于尽`，不公开普通棋子的具体军衔。

### 终局

- 夺取对方军旗的一方获胜。
- 当前方没有任何可移动棋子或合法走法时判负；若双方均无可移动棋子则判和棋。
- 连续 31 个半回合没有发生碰子时，完成第 31 个无碰子行动的一方判负，对应竞赛棋谱 `SL[31]` 约束。
- 和棋不改变比分、下一局玩家执方或智能等级。

## 暗棋信息模型

### 真实状态

`JunqiState` 保存完整棋盘、棋子 ID、真实类型、阵营、是否移动、军旗公开状态、当前方、连续不碰子计数和终局结果。只有规则裁决层可以读取双方真实类型。

### 观察状态

`JunqiObservation` 面向一个指定观察者生成：

- 己方存活棋子显示真实类型。
- 敌方普通棋子只显示稳定棋子 ID、位置、是否移动及可见约束，不显示真实类型。
- 司令阵亡后，敌方军旗显示为军旗。
- 已移出棋盘的敌方棋子只记录碰子结果，不暴露其真实类型。
- UI 和 AI 都消费观察状态；禁止从 `JunqiState` 直接构造 AI 搜索输入。

### 知识更新

`JunqiKnowledge` 为每个敌方存活棋子保存候选类型集合：

- 棋子一旦移动，候选集合移除军旗和地雷。
- 军旗公开后对应棋子的候选集合缩为军旗。
- 碰子后根据己方已知棋子类型、进攻/防守方向和裁决结果约束存活敌子的候选集合。
- 全局候选计数不得超过初始棋子数量；确定化采样允许未分配类型代表已阵亡但身份未知的棋子。

## 单人 AI

### 公平性边界

- `JunqiAi.chooseMove` 只接受 `JunqiObservation`、AI 己方真实棋子和 `JunqiKnowledge`。
- 搜索测试构造两份观察完全相同但敌方真实身份不同的局面，AI 必须给出相同候选集合和确定性选择。
- 随机源由局面观察哈希、等级和固定包版本盐共同生成，不使用真实敌方类型。

### 搜索方法

- 先生成全部合法候选行动并执行战术优先级：立即夺旗、阻止已公开军旗被夺、工兵排已确认地雷、炸弹交换高概率大子。
- 对未知敌方棋子按候选集合和剩余数量做一致性确定化采样。
- 每个样本执行有限深度 Negamax/Alpha-Beta 搜索，汇总每个根行动的平均分和最差分。
- 评估包括军旗安全、夺旗距离、存活军阶价值、工兵与地雷结构、炸弹交换潜力、铁路控制、行营控制、机动性和连续不碰子风险。
- 搜索在 Compose UI 线程之外运行，返回前校验请求世代与当前状态，丢弃过期结果。

### 1-10 级梯度

单人等级为玩家累计胜局数加一，上限 10。每级配置集中在不可变表中：

| 等级 | 样本数 | 深度 | 节点预算 | 时间预算 |
| ---: | ---: | ---: | ---: | ---: |
| 1 | 1 | 1 | 100 | 80 ms |
| 2 | 2 | 1 | 250 | 100 ms |
| 3 | 3 | 2 | 700 | 150 ms |
| 4 | 4 | 2 | 1,500 | 220 ms |
| 5 | 6 | 2 | 3,000 | 320 ms |
| 6 | 8 | 3 | 7,000 | 480 ms |
| 7 | 10 | 3 | 14,000 | 650 ms |
| 8 | 14 | 3 | 28,000 | 850 ms |
| 9 | 18 | 4 | 55,000 | 1,100 ms |
| 10 | 24 | 4 | 100,000 | 1,400 ms |

低等级从前若干候选中做确定性弱化；高等级选择汇总分最高行动。节点和时间任一预算耗尽时返回最后完成的根行动评估。

## 会话流程

### 阶段

`JunqiPhase` 包含：

1. `DEPLOYMENT`：当前方布阵。
2. `HANDOFF`：全屏不透明遮罩，只显示下一方名称和接管按钮。
3. `PLAYING`：当前方查看己方信息并行动。
4. `BATTLE_RESULT`：保留行动方可见的观察态棋盘和最后一步，在侧栏显示不含军衔的碰子裁决，确认后进入下一方交接或机器人回合。
5. `FINISHED`：显示胜负或和棋与重新开始。

### 视角与隐私

- 当前人类阵营始终显示在棋盘下方；蓝方或双人下一方接管时棋盘做 180 度坐标映射，棋子文字保持正向。
- `HANDOFF` 使用完全不透明背景，离开前一帧不得包含棋盘或军衔语义节点。
- `BATTLE_RESULT` 只能发布当前观察者本就可见的 `JunqiObservation`、最后一步和通用裁决；敌方棋子继续只有稳定 ID、位置、移动状态与公开约束，不能发布真实军衔。
- 双人模式每次行动后进入交接；单人模式只在玩家执蓝的新局开始前需要一次接管确认，AI 回合不展示 AI 私有观察。

### 比分、重开与悔棋

- 单人比分按玩家与智能身份记录，智能等级由玩家胜局决定；双人比分按红方与蓝方记录。
- 单人玩家胜利后重新开始交换执方；玩家失败后恢复红方先手；和棋保持当前执方。
- 双人重新开始固定红方先手，并重新进行双方布阵。
- 单人悔棋恢复玩家合法行动之前的快照，并回退随后的 AI 行动、知识状态、比分、连续不碰子计数和最后移动。
- 双人悔棋回退一步，之后进入应行动方的遮屏交接。
- 产生胜者后隐藏悔棋；和棋保留悔棋。重新开始清空历史。

## 界面与视觉

### 页面结构

游戏仍只实现 `MainScreen` 和 `GameScreen`：

- `MainScreen`：完整棋盘预览、圆形军棋 Logo、标题 `军棋`、版本号及三个统一菜单按钮。
- `GameScreen`：同一个页面内部切换布阵、交接、对局、碰子结果和终局阶段。

横屏平板以棋盘为主要区域，右侧为固定信息与操作栏。紧凑宽度仍保持棋盘和右栏并排，必要时缩小棋盘，不把棋盘放进装饰卡片。

### 视觉方向

- 明亮现代作战地图，而非暗色战争游戏大厅。
- 背景使用冷矿物灰 `#E9EFEC`，棋盘使用浅灰白 `#F7F6F0`。
- 公路使用石墨绿 `#536762`，铁路使用松针绿 `#214F45`，行营使用浅青绿 `#DCEAE3`，大本营使用低饱和黄铜 `#C9B779`。
- 红方使用朱红 `#B83A32`，蓝方使用靛蓝 `#315B83`，界面主墨色 `#20312E`。
- 最后移动沿用所有游戏共享的半透明亮蓝色四角框：比例 `0.92`、内缩 `0.04`、角长 `0.18`、高亮 `0xB84FCBFF`、阴影 `0x70115C93`。
- 棋子为带小型肩章刻线的圆角矩形军棋牌。己方牌显示军衔，敌方牌只显示阵营色棋背和星形标记。

### 包内素材

- `assets/icon.png`：1024x1024 圆形安全区 Logo，两枚对向军棋牌围绕铁路交汇点，不含小字。
- `assets/board/junqi-board.png`：1400x1680 RGBA 完整棋盘底图，包含公路、铁路、行营、大本营和中部边界，不包含棋子。
- `assets/textures/junqi-shelf.png`：开始界面的浅浮雕陈列纹理。
- 素材由 `games/junqi/tools/generate_junqi_assets.py` 可重复生成；运行时按文档化网格边界注册点击坐标，素材无效时回退到 Compose 绘制。

## 测试与验收

### 单元测试

- 清单版本、统一菜单和版本标签。
- 60 节点类型、全部公路边、铁路边、大本营和行营坐标。
- 默认、随机、交换布阵的棋子数量和全部限制。
- 公路一步、普通棋子铁路直行、工兵铁路转弯、阻挡、行营保护、大本营锁定。
- 全部军阶、同级、炸弹、地雷、工兵、军旗和司令明旗碰子矩阵。
- 夺旗、单方无棋可动、双方无棋可动和 `SL=31` 终局。
- 观察状态不泄露敌方军衔，移动与碰子正确收窄知识集合。
- 两个真实身份不同但观察相同的局面产生相同 AI 结果。
- AI 立即夺旗、阻止夺旗、排雷、炸弹交换和预算单调性。
- 单人/双人布阵、交接、异步 AI、悔棋、重开、比分、执方交换和最后移动。
- Logo、棋盘 PNG 的尺寸、透明角、包内路径和注册坐标。

### 集成验收

- `./gradlew :games:junqi:testDebugUnitTest` 通过。
- `npm run build:game:junqi` 生成合法 `junqi.zip`。
- `npm run verify` 验证五个内置包及 Debug APK 内的 `assets/builtin-games/junqi.zip`。
- 模拟器验证主界面自动显示包提供的 `军棋` 名称与 Logo，不增加游戏 ID 展示分支。
- 在 800x600 dp 与更大横屏视口验证菜单、布阵、红蓝交接、单人对局、双人对局、碰子结果、最后移动、悔棋和终局均无重叠。

## 非目标

- 不实现四国军棋、联网匹配、排行榜、聊天、残局库或在线更新。
- 不让 AI 读取敌方真实军衔，不以全知搜索冒充暗棋智能。
- 不修改 `game-api` 或 `CURRENT_SHELL_API`。
- 不引入外部棋类引擎、服务器或运行时网络依赖。
