# Design Brief - 二国军棋界面重设计

**Slug:** `junqi-ui`
**User brief (verbatim summary):** 使用 `ai-everything:designer` 重新设计二国军棋，棋盘参考用户提供的传统二国军棋盘图片；棋子改为绿色/橙色底，军衔使用尽量大的白色黑体。
**Stack:** Android Kotlin + Jetpack Compose，离线动态游戏包，横屏 Android Pad
**Iteration:** 2026-07-22T22:14:07+08:00
**Status:** 用户已选择方案 1；本文件是军棋 `0.0.10` 已实现界面的视觉 SSOT。

## Base System

`ui-ux-pro-max` 将产品识别为触屏娱乐与棋盘游戏，强调 48dp 触控目标、8dp 控件间距、明确按压反馈、横屏适配和不依赖颜色的状态表达。其自动推荐的深色 OLED、蓝橙 CTA 和 Roboto 单字体过于通用，且与军棋的实体棋具语义不够贴合，因此仅保留交互与可访问性约束。

| Dimension | Content |
| --- | --- |

| Page structure | 左侧固定棋盘主舞台，右侧固定军令栏；菜单页、布阵、对局、交接、碰子、终局共享同一横屏空间逻辑 |

## Revised Direction

### Three approaches considered

1. **传统铁路军棋盘加高辨识军牌，推荐。** 直接吸收参考图的麦秆金盘面、矩形兵站、椭圆行营、黑白枕木铁路、中央三路桥和红色大本营字样；棋子以深绿和深橙整面区分阵营，白色粗黑体军衔成为最醒目的棋子信息。
2. **深绿指挥沙盘。** 墨绿台面与黄铜铁路具有高级棋具感，但偏离用户给出的传统棋盘辨识，且行营、兵站和铁路语义不如参考图直接。
3. **现代战术 HUD。** 黑底、网格、荧光轨迹能突出暗棋情报，但更像射击或雷达游戏，弱化了军棋作为实体棋盘游戏的触感，也不适合家庭平板的明亮环境。

### Detemplating answers

1. **Subject grounding** - 具体对象是同一台 Android Pad 上进行的二国暗棋；受众是需要快速辨认棋路、己方军衔与当前交接方的棋类玩家；单屏任务是完成一次布阵或走棋，而不是展示功能。
2. **Memory point (signature)** - 黑白相间的枕木铁路和中央三路桥。它来自传统军棋盘真实走法结构，缩略时仍能一眼识别军棋，而不是一般连线棋盘。
3. **Aesthetic risk** - 在数字界面中保留 `兵站`、`行营`、`大本营` 的盘面文字，并让空节点像印刷棋盘一样可读。文字不是装饰，而是在棋子移开后继续解释节点类型。
4. **Detemplating changes** - 删除蓝橙 SaaS 配色、深绿黄铜西式棋具感和右侧普通卡片；改为麦秆金纤维盘面、墨色公路、黑白铁路、传统节点字样、绿色/橙色方形军牌，以及无外框的军令簿式信息栏。
5. **Rejected defaults** - 参考图允许使用暖金盘面，但拒绝奶油色加陶土红的生活方式模板、近黑加荧光单色、报纸栏目、迷彩、弹孔、枪械和战争海报式装饰。

### Subject-specific decisions

- `designs/references/junqi-classic-board-reference.jpg` 是本轮棋盘形制参考，只提取结构、线型、节点形状和温和金色调，不复制水印或低清纹理。
- 棋盘表面为低饱和麦秆金纤维板，外圈使用灰绿包边；不出现地形照片或迷彩。
- 公路是墨色单线；铁路使用深墨与象牙交替的枕木带，铁路必须比公路更醒目。中央保留三条独立桥接，不合并为一条装饰横梁。
- 兵站为细边矩形并标 `兵站`；行营为浅灰绿椭圆并标 `行营`；大本营为矩形并使用红色 `大本营`。节点类型同时通过形状和文字区分。
- 两方棋子均为整面阵营色方牌：绿方 `#23704B`，橙方 `#C65012`。明棋军衔使用纯白无衬线黑体，暗棋只保留白色简化五角星或盾徽，不显示军衔。
- 军衔文字在保留四周至少 6% 内边距的前提下占棋子高度约 62%-68%，使用 `FontWeight.Black`；两字标签不换行、不压缩字距、不增加描边或阴影。
- 棋子占单元格宽度约 70%、行距约 58%，保持铁路和相邻棋子之间的清晰间隔。
- 共享最后移动标记继续使用半透明亮蓝四角框，位于棋子外侧且不遮挡军衔。
- 右栏不做悬浮卡片。顶部比分、中央回合军令、底部操作被两条细灰绿规则线组织成一张平整军令簿。

## Typography

- Display: Android `FontFamily.Serif`，等价于 Noto Serif CJK，`军棋` 48sp / 600，短标题 26sp / 600。
- Body: Android `FontFamily.SansSerif`，18sp / 500；次级信息 14sp / 400。
- Utility: Android `FontFamily.Monospace`，比分 38sp / 500、等级和版本 13-15sp。
- Piece rank: Android `FontFamily.SansSerif` + `FontWeight.Black`（900），纯白，800x600dp 基线约 18-21sp；优先放大到棋子高度 62%-68%，仅在触边时等比缩小。
- 字距固定为 0；按钮与棋子标签不使用负字距；军衔不换行、不使用衬线、描边或阴影。

## Layout Concept

“一张被数字化保存的传统铁路军棋盘，旁边夹着一页现代军令簿。”棋盘不是装饰卡片，右栏也不再是独立悬浮面板；经典棋盘负责识别，现代右栏负责触控效率。

### Main menu, Xiangqi-reference landscape baseline

```text
┌──────────────────────────────┬──────────────────┐
│                              │    package logo  │
│  classic railway board       │       军棋       │
│  clean, unoccupied topology  │   版本 0.0.10    │
│                              │                  │
│╫╫ three striped bridges ╫╫╫╫│──────────────────│
│                              │   [单人模式]     │
│                              │   [双人对战]     │
│                              │   [退出游戏]     │
└──────────────────────────────┴──────────────────┘
```

- 宽布局使用象棋参考几何：页面外边距 28dp、棋盘与右栏间距 34dp；菜单右栏固定 320dp 宽并占内容高度 88%，棋盘在剩余区域居中并保持 1400:1680 比例。
- Logo 与标题只占右栏上半部，不使用大面积纹理卡片。
- 三个菜单按钮固定 64dp 高、8dp 圆角以内；主按钮为灰绿实心，次按钮为浅面描边，退出为红色文字描边。

### Game screen

```text
┌──────────────────────────────┬──────────────────┐
│  green hidden backs          │ 玩家       智能 │
│                              │   0   :   0      │
│╫╫ three striped bridges ╫╫╫╫│──────────────────│
│  orange square pieces        │ 橙方回合         │
│  selected + legal markers    │ 玩家执橙 · 等级1 │
│                              │                  │
│                              │ [悔棋] [返回菜单]│
└──────────────────────────────┴──────────────────┘
```

- 对局右栏固定 300dp 宽并占内容高度 94%；棋盘完整显示 12x5 节点，不允许滚动、裁切或遮挡。
- 比分使用绿橙短色条加文字，数字使用等宽字体；当前回合采用 4dp 阵营色左边线和完整文字，不做药丸标签。
- 布阵阶段底部操作为 `随机`、`重置` 两个次操作和整行主操作 `出战`。
- 对局阶段只显示当前可用操作；终局隐藏悔棋并以 `重新开始` 作为唯一主操作。

## State Screens

- `HANDOFF`：完全不透明的灰绿织纹面；中央仅显示下一方阵营方牌、`请绿方接管` 或 `请橙方接管` 和 `接管棋盘`。不得显示棋盘轮廓、棋子、比分或最后移动。
- `BATTLE_RESULT`：保留当前观察者可见棋盘和最后一步，右栏状态区显示通用结果，操作区显示 `确认判定`。不得显示敌方军衔；确认后双人模式立即进入下一方不透明交接页。
- `FINISHED`：棋盘仍保留最终局面；右栏中央显示胜负或和棋，底部只保留 `重新开始`、`返回菜单`，胜负状态不显示悔棋。
- Robot thinking：回合文字改为 `智能思考中`，棋盘触控禁用；最后移动标记保留，不增加全屏加载遮罩。

## Interaction

- 所有按钮最小 48dp，高优先级按钮 56-64dp；按钮间距至少 8dp。
- 棋子按下时在 80ms 内降低 4% 明度并下沉 1dp；选中后显示灰绿内框和合法落点，不能仅依赖缩放。
- 合法落点使用墨色圆点，攻击目标使用黑白相间断环；最后移动仍只使用亮蓝四角框。
- 页面切换只允许 160-220ms 的淡入和轻微位移；交接页立即遮蔽前一帧，不使用透明过渡。
- 支持系统字体缩放；文本不能覆盖棋盘或按钮。读屏顺序先棋盘状态，再比分/回合，最后操作。

## Copy Tone

- Register: 简短、明确、像一张军令，不写故事旁白或功能说明。
- Vocabulary: 使用玩家直接控制的词，例如 `出战`、`接管棋盘`、`悔棋`、`重新开始`。
- Empty/error state: 说明原因和下一步，例如 `当前阵型不符合布阵规则，请调整军旗、地雷或炸弹位置`。

## Implementation Notes

- Primary CTA labels: 菜单 `单人模式`；布阵 `出战`；交接 `接管棋盘`；终局 `重新开始`。
- Components: `JunqiClassicBoard`, `JunqiSquarePiece`, `JunqiOrderRail`, `JunqiScoreLine`, `JunqiTurnOrder`, `JunqiActionButton`, opaque handoff/result screens.
- Package assets: 按参考形制重做完整棋盘 PNG、菜单陈列纹理和军棋 Logo；棋子保持 Compose 绘制或拆分为红蓝正反面 PNG，最终以清晰缩放和包体积评估决定。
- Preserve: 12x5 几何、点击映射、观察态隐私、共享最后移动框、象棋参考棋盘/侧栏布局、缺图 fallback。
- Non-goals: 不改变规则、AI、公平性模型、比分/悔棋/重开逻辑、游戏中心外壳或动态包协议。

## Delivery Guardrails

- 正常文字对比至少 4.5:1，大标题至少 3:1。
- 白色军衔与绿方棋子对比约 6.02:1，与橙方棋子对比约 4.61:1，均按正常文字 AA 标准验收。
- 不出现嵌套卡片、胶囊标签、渐变光球、迷彩、枪械、弹孔、徽章堆砌或装饰性编号。
- 页面上只保留一个视觉主操作；其余操作降级为描边或文本按钮。
- 任何纹理都必须服从路网和军衔可读性，棋盘缩小到 800x600dp 时仍能看清铁路、公路、行营和大本营。
