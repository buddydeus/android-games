# Design Brief - 二国军棋界面重设计

**Slug:** `junqi-ui`
**User brief (verbatim summary):** 使用 `ai-everything:designer` 重新设计二国军棋。
**Stack:** Android Kotlin + Jetpack Compose，离线动态游戏包，横屏 Android Pad
**Iteration:** 2026-07-22T21:31:11+08:00
**Status:** 设计方向待用户选择；本文件是本轮预览与后续实现的视觉 SSOT。

## Base System

`ui-ux-pro-max` 将产品识别为触屏娱乐与棋盘游戏，强调 48dp 触控目标、8dp 控件间距、明确按压反馈、横屏适配和不依赖颜色的状态表达。其自动推荐的深色 OLED、蓝橙 CTA 和 Roboto 单字体过于通用，且与军棋的实体棋具语义不够贴合，因此仅保留交互与可访问性约束。

## Revised Direction

### Three approaches considered

1. **指挥沙盘与军令簿，推荐。** 明亮冷灰空间包围一张深墨绿实体沙盘，黄铜铁路和红蓝珐琅军牌形成真实棋具感。识别强、长时间对局舒适，素材也可由游戏包独立提供。
2. **旧式军务档案地图。** 泛黄纸张、红蓝铅笔线和印章语汇很贴题，但容易落入奶油色加复古装饰的常见模板，且铁路、公路与棋子文字对比不足。
3. **现代战术 HUD。** 黑底、网格、荧光轨迹能突出暗棋情报，但更像射击或雷达游戏，弱化了军棋作为实体棋盘游戏的触感，也不适合家庭平板的明亮环境。

### Detemplating answers

1. **Subject grounding** - 具体对象是同一台 Android Pad 上进行的二国暗棋；受众是需要快速辨认棋路、己方军衔与当前交接方的棋类玩家；单屏任务是完成一次布阵或走棋，而不是展示功能。
2. **Memory point (signature)** - 横贯棋盘中央的“黄铜铁路桥脊”：双线铁路在中央收束为带铆钉的金属桥，红蓝阵地在此对峙。它编码真实棋盘结构，不是装饰分割线。
3. **Aesthetic risk** - 棋盘主体使用大面积深墨绿，而应用外壳保持明亮冷灰。深色只花在核心棋盘上，既提升象牙路网与黄铜铁路的辨识度，也避免整个界面变成泛化暗色游戏大厅。
4. **Detemplating changes** - 删除蓝橙 SaaS 配色、单一 Roboto 和右侧普通卡片；改为墨绿沙盘、骨白路网、黄铜铁路、红蓝珐琅军牌，以及无外框的军令簿式信息栏。
5. **Rejected defaults** - 明确拒绝奶油色加陶土红、近黑加荧光单色、报纸细线栏目三类默认组合；也拒绝迷彩、弹孔、枪械和战争海报式装饰。

### Subject-specific decisions

- 棋盘表面像细颗粒军用指挥台，不出现地形照片或迷彩，保证路网可读。
- 公路是骨白单线；铁路是黄铜双轨并带离散枕木；铁路必须比公路更醒目。
- 行营为低饱和青灰圆形凹位，大本营为黄铜盾形凹位，节点类型同时通过形状和颜色区分。
- 己方棋子为象牙珐琅面、阵营色边框和顶部肩章刻线；敌方棋背为实心红或蓝珐琅面、中央浅浮雕盾徽，不显示军衔。
- 棋子占单元格宽度约 70%、行距约 58%，保持铁路和相邻棋子之间的清晰间隔。
- 共享最后移动标记继续使用半透明亮蓝四角框，位于棋子外侧且不遮挡军衔。
- 右栏不做悬浮卡片。顶部比分、中央回合军令、底部操作被两条细黄铜规则线组织成一张平整军令簿。

## Typography

- Display: Android `FontFamily.Serif`，等价于 Noto Serif CJK，`军棋` 48sp / 600，短标题 26sp / 600。
- Body: Android `FontFamily.SansSerif`，18sp / 500；次级信息 14sp / 400。
- Utility: Android `FontFamily.Monospace`，比分 38sp / 500、等级和版本 13-15sp。
- 字距固定为 0；按钮与棋子标签不使用负字距；军衔为 16-18sp / 600，随棋盘缩放但不低于 13sp。

## Layout Concept

“一张占据视野的实体指挥沙盘，旁边夹着一页军令簿。”棋盘不是卡片，右栏也不再是独立悬浮面板；二者由中央战线与黄铜规则线形成稳定的左右关系。

### Main menu, 800x600dp baseline

```text
┌──────────────────────────────┬──────────────────┐
│                              │    package logo  │
│  full-height command board   │       军棋       │
│  clean, unoccupied topology  │    版本 0.0.8    │
│                              │                  │
│════ brass center bridge ═════│──────────────────│
│                              │   [单人模式]     │
│                              │   [双人对战]     │
│                              │   [退出游戏]     │
└──────────────────────────────┴──────────────────┘
```

- 棋盘约占可用宽度 64%，右栏约占 30%，中间保留 24dp 空隙。
- Logo 与标题只占右栏上半部，不使用大面积纹理卡片。
- 三个菜单按钮固定 64dp 高、8dp 圆角以内；主按钮为墨绿实心，次按钮为浅面描边，退出为红色文字描边。

### Game screen

```text
┌──────────────────────────────┬──────────────────┐
│  blue enamel backs           │ 玩家       智能 │
│                              │   0   :   0      │
│════ riveted brass bridge ════│──────────────────│
│  ivory/red own pieces        │ 红方回合         │
│  selected + legal markers    │ 玩家执红 · 等级1 │
│                              │                  │
│                              │ [悔棋] [返回菜单]│
└──────────────────────────────┴──────────────────┘
```

- 棋盘在 800x600dp 仍完整显示 12x5 节点，不允许滚动、裁切或遮挡。
- 比分使用红蓝短色条加文字，数字使用等宽字体；当前回合采用 4dp 阵营色左边线和完整文字，不做药丸标签。
- 布阵阶段底部操作为 `随机`、`重置` 两个次操作和整行主操作 `出战`。
- 对局阶段只显示当前可用操作；终局隐藏悔棋并以 `重新开始` 作为唯一主操作。

## State Screens

- `HANDOFF`：完全不透明的深墨绿织纹面；中央仅显示下一方阵营盾牌、`请蓝方接管` 或 `请红方接管` 和 `接管棋盘`。不得显示棋盘轮廓、棋子、比分或最后移动。
- `BATTLE_RESULT`：同一不透明面，中央使用两块闭合军牌碰合的抽象压印图形；只显示通用结果和 `确认`。
- `FINISHED`：棋盘仍保留最终局面；右栏中央显示胜负或和棋，底部只保留 `重新开始`、`返回菜单`，胜负状态不显示悔棋。
- Robot thinking：回合文字改为 `智能思考中`，棋盘触控禁用；最后移动标记保留，不增加全屏加载遮罩。

## Interaction

- 所有按钮最小 48dp，高优先级按钮 56-64dp；按钮间距至少 8dp。
- 棋子按下时在 80ms 内降低 4% 明度并下沉 1dp；选中后显示黄铜内框和合法落点，不能仅依赖缩放。
- 合法落点使用骨白圆点，攻击目标使用阵营无关的黄铜断环；最后移动仍只使用亮蓝四角框。
- 页面切换只允许 160-220ms 的淡入和轻微位移；交接与碰子页立即遮蔽前一帧，不使用透明过渡。
- 支持系统字体缩放；文本不能覆盖棋盘或按钮。读屏顺序先棋盘状态，再比分/回合，最后操作。

## Copy Tone

- Register: 简短、明确、像一张军令，不写故事旁白或功能说明。
- Vocabulary: 使用玩家直接控制的词，例如 `出战`、`接管棋盘`、`悔棋`、`重新开始`。
- Empty/error state: 说明原因和下一步，例如 `当前阵型不符合布阵规则，请调整军旗、地雷或炸弹位置`。

## Implementation Notes

- Primary CTA labels: 菜单 `单人模式`；布阵 `出战`；交接 `接管棋盘`；终局 `重新开始`。
- Components: `JunqiCommandBoard`, `JunqiEnamelPiece`, `JunqiOrderRail`, `JunqiScoreLine`, `JunqiTurnOrder`, `JunqiActionButton`, opaque handoff/result screens.
- Package assets: 重做完整棋盘 PNG、菜单陈列纹理和军棋 Logo；棋子保持 Compose 绘制或拆分为红蓝正反面 PNG，最终以清晰缩放和包体积评估决定。
- Preserve: 12x5 几何、点击映射、观察态隐私、共享最后移动框、800x600dp 并排布局、缺图 fallback。
- Non-goals: 不改变规则、AI、公平性模型、比分/悔棋/重开逻辑、游戏中心外壳或动态包协议。

## Delivery Guardrails

- 正常文字对比至少 4.5:1，大标题至少 3:1。
- 不出现嵌套卡片、胶囊标签、渐变光球、迷彩、枪械、弹孔、徽章堆砌或装饰性编号。
- 页面上只保留一个视觉主操作；其余操作降级为描边或文本按钮。
- 任何纹理都必须服从路网和军衔可读性，棋盘缩小到 800x600dp 时仍能看清铁路、公路、行营和大本营。
