# 国际象棋

国际象棋是游戏中心内置的离线游戏包，提供完整标准走法、自动和棋判断、1-10 级本地搜索 AI、单人机器人对战和本地双人对战。规则、搜索、会话状态、界面和棋子素材全部归本模块所有。

## 当前玩法

- 支持王、后、车、象、马和兵的完整合法走法，并过滤会使己方王处于将军状态的移动。
- 支持王车易位、吃过路兵，以及升变为后、车、象或马；未显式选择时规则层默认升变为后。
- 支持将军、将死、逼和、三次重复、五十回合规则和子力不足和棋。
- 将死判断优先于自动和棋判断。
- 单人玩家执黑时棋盘旋转 180 度，黑方位于下方，棋子保持正向。
- 最后移动的目标格使用半透明亮蓝色四角框标记。
- 所有和棋都不改变比分、下一局执方或智能等级。

## 单人 AI

单人智能等级等于玩家累计胜局数加一，上限为 10 级。AI 完全离线运行，按等级逐步增加深度、节点预算、思考时间、评估项和静态搜索范围。

核心实现包括：

- 迭代加深 Negamax 与 Alpha-Beta 剪枝。
- 原地 make/unmake 搜索局面。
- 有界置换表、杀手着法与走法排序。
- 棋子位置、活动度和完整局面评估。
- 和棋感知静态搜索，以及包含完整重复次数上下文的搜索键。
- 搜索与会话共用等价的重复局面键，包括规范化的吃过路兵权利。

玩家默认执白先手。玩家获胜后重新开始会交换执方；玩家失败后恢复执白先手；和棋保持当前执方。玩家成为黑方时，白方机器人先行。

## 悔棋、比分与终局

- 单人模式一次悔棋回退玩家移动及随后的机器人应答。
- 双人模式一次悔棋回退一步。
- 快照同时恢复棋盘、重复局面计数、比分、终局结果和最后移动。
- 产生胜者后隐藏悔棋；和棋保留悔棋。
- 单人比分按玩家与机器人身份记录，双人比分按白方与黑方记录。
- 终局右栏显示精确结果并提供 `重新开始`。

## 棋子素材与界面

- `package/assets/icon.png` 是游戏中心读取的 1024x1024 圆形安全区 Logo。
- `package/assets/pieces/` 包含黑白双方共 12 张独立的 1024x1024 透明 Staunton 棋子 PNG。
- 菜单棋盘、对局棋盘和升变选择器按当前游戏包版本加载并缓存贴图。
- 运行时只裁去完全透明的外边距，棋子按棋盘格的 82% 显示。
- 单张贴图缺失或无效时，回退到对应 Unicode 棋子。

视觉规范见 [国际象棋 UI 设计](../../designs/specs/international-chess-ui.md)，规则与搜索边界见 [国际象棋游戏设计](../../docs/superpowers/specs/2026-07-18-international-chess-game-design.md)。

## 代码结构

| 路径 | 作用 |
| --- | --- |
| `ChessRules.kt` | 完整合法走法、特殊走法、将军、胜负与和棋判断 |
| `ChessAi.kt` | 1-10 级配置和 AI 入口 |
| `ChessSearchPosition.kt` | 原地 make/unmake 搜索局面 |
| `ChessSearchEngine.kt` | 迭代加深、剪枝、排序、置换表和静态搜索 |
| `ChessSession.kt` | 比分、执方、重复计数、悔棋快照、升变和视角映射 |
| `ChessPieceTextures.kt` | 包内棋子贴图加载与透明边距裁切 |
| `ChessVisuals.kt` | 开始界面、棋盘、升变选择、右栏和状态覆盖层 |
| `ChessPlugin.kt` | `GamePlugin` 入口、异步机器人和对局编排 |

## 构建与测试

在仓库根目录执行：

```bash
./gradlew :games:chess:testDebugUnitTest
npm run build:game:chess
npm run verify
```

测试覆盖特殊走法、公开状态的合法性、perft 局面、将死与和棋优先级、重复局面键、会话重开与悔棋、AI 等级预算和棋子素材约束。

## 更新要求

修改国际象棋规则、AI、界面或包内素材时，需要同步递增插件清单与 `package/manifest.json` 中的 `versionCode`、`versionName`，更新对应测试和本文档。仅修改文档不递增游戏版本。通用游戏包契约见 [游戏插件开发说明](../../docs/agents/game-plugins.md)。
